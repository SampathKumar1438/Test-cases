Retainful Test Cases

Positive Test Cases

  1. User can successfully sign up with valid email and password
  2. User receives a confirmation email after signing up.
  3. User can log in with valid credentials.
  4. User can reset the password using a valid email address.
  5. User can access the dashboard after logging in.
  6. User can create a new abandoned cart email campaign.
  7. User can customize an email template using the drag-and-drop editor.
  8. User can save changes to an email template.
  9. User can schedule an email campaign to be sent at a later date.
  10. User can view analytics for an email campaign.
  11. User can segment customers based on purchase history.
  12. User can import contacts from a CSV file.
  13. User can export customer data to a CSV file.
  14. User can set up automated email sequences.
  15. User can integrate with WooCommerce.
  16. User can integrate with Shopify.
  17. User can configure and use dynamic coupons in emails.
  18. User can view a list of all abandoned carts.
  19. User can see detailed information about a specific abandoned cart.
  20. User can see the open and click rates for sent emails.
  21. User can add new team members to the account.
  22. User can assign different roles to team members.
  23. User can remove team members from the account.
  24. User can customize the "from" email address for campaigns.
  25. User can set up a sending domain.
  26. User can enable GDPR compliance settings.
  27. User can create and manage signup forms.
  28. User can preview emails before sending them.
  29. User can test emails by sending them to a specific address.
  30. User can enable two-factor authentication for security.
  31. User can view the pricing plans and select one.
  32. User can upgrade or downgrade their subscription plan.
  33. User can access the help center and support resources.
  34. User can submit a support ticket.
  35. User can view FAQ and knowledge base articles.
  36. User can follow a step-by-step guide for integrating with a platform.
  37. User can view past invoices and billing history.
  38. User can update payment method information.
  39. User receives a notification when a new feature is released.
  40. User can connect Retainful with a third-party email service provider like MailChimp.

Negative Test Cases

  1. User cannot sign up with an invalid email address.
  2. User cannot sign up with a weak password (e.g., "12345").
  3. User does not receive a confirmation email after signing up.
  4. User cannot log in with incorrect credentials.
  5. User cannot reset the password with an invalid email address.
  6. User is redirected to an error page when accessing the dashboard without logging in.
  7. User cannot save changes to an email template due to a server error.
  8. User cannot schedule an email campaign without specifying a date and time.
  9. User sees incorrect analytics data for an email campaign.
  10. User cannot segment customers due to missing segmentation criteria.
  11. User cannot import contacts from a malformed CSV file.
  12. User cannot export customer data due to a permission error.
  13. User cannot set up automated email sequences without completing all required fields.
  14. User encounters an error while integrating with WooCommerce.
  15. User encounters an error while integrating with Shopify.
  16. Dynamic coupons do not appear correctly in emails.
  17. User cannot view the list of abandoned carts due to a loading error.
  18. User sees incorrect details for a specific abandoned cart.
  19. Open and click rates for sent emails are not displayed correctly.
  20. User cannot add new team members due to a permission error.
  21. User cannot assign roles to team members due to a missing role list.
  22. User cannot remove team members due to a server error.
  23. User cannot customize the "from" email address due to invalid format.
  24. User cannot set up a sending domain due to DNS configuration errors.
  25. GDPR compliance settings are not saved correctly.
  26. User cannot create a signup form due to missing fields.
  27. Email preview does not render correctly.
  28. Test email is not received by the specified address.
  29. Two-factor authentication setup fails.
  30. Pricing plans page does not load.
  31. User cannot upgrade or downgrade their subscription plan due to a payment error.
  32. User cannot access the help center due to a page load error.
  33. Support ticket submission fails.
  34. FAQ and knowledge base articles do not load.
  35. Step-by-step integration guide contains broken links.
  36. Past invoices and billing history are not displayed correctly.
  37. Payment method update fails.
  38. User does not receive notifications for new feature releases.
  39. Connection to a third-party email service provider like MailChimp fails.
  40. User sees generic error messages instead of specific error details.
